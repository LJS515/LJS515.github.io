人形机器人中的状态估计：IMU、足底接触与浮动基状态恢复

到这里，人形这条主线已经把几个关键层都铺开了：

  • 质心、ZMP、步态
  • 全身控制
  • QP 求解
  • 接触力分配与支撑约束

但如果你真的把这些模块接进一个系统里，很快会发现还有一个决定成败的核心环节：

状态估计。

因为控制器再完整，QP 再漂亮，接触力分配再合理，它们都建立在一个前提上：

系统对“自己当前在什么状态”有足够可靠的估计。

如果这个前提不成立，后面的所有控制都可能建立在错误输入上。

这就是为什么人形机器人里，经常会出现一种非常典型的错觉：

  • 看起来像控制器不稳
  • 看起来像步态参数不对
  • 看起来像支撑切换有问题

但最后往下追，根子却常常在状态估计。

下面专门把这条链讲清楚。

一、为什么人形机器人特别依赖状态估计

机械臂很多时候固定在底座上，世界和本体关系相对简单。

人形不是。

人形控制的是一个浮动基系统，而且它还在持续接触切换。

1.1 这意味着什么

意味着机器人必须不断估计：

  • 躯干姿态
  • 躯干角速度
  • 浮动基线速度
  • 浮动基位置
  • 哪只脚在支撑
  • 支撑脚和世界的关系

1.2 为什么这些量不能直接全测出来

因为真实系统里没有一个传感器能直接告诉你：

  • 机器人骨盆在世界坐标系里的完整状态
  • 每个时刻哪只脚真的“可靠接地”
  • 当前整机速度到底是多少

你拿到的通常只是局部测量：

  • IMU
  • 关节编码器
  • 足底力传感器或接触开关
  • 有时还有视觉或外部定位

然后你必须把这些局部观测拼起来，恢复整机状态。

1.3 一个更准确的说法

人形控制并不是“先知道状态，再做控制”。

真实系统更像是：

在不完全、带噪声、会漂移的观测里，持续估计一个足够能用于控制的内部状态。

二、什么是浮动基状态

这几个字在做人形时一定要真正搞懂。

2.1 为什么叫浮动基

因为人形机器人没有固定底座。

它的主刚体，比如骨盆或躯干，在世界里本来就是自由运动的。

所以从建模角度看，这个“基座”本身不是固定参考，而是一个需要被估计的自由刚体。

2.2 浮动基状态通常包含什么

最常见的一组量包括：

  1. 位置
  2. 姿态
  3. 线速度
  4. 角速度

这四类量决定了整机主刚体在世界中的基本状态。

2.3 为什么这些量这么关键

因为后面的很多核心控制量都依赖它：

  • 质心位置和速度
  • 躯干姿态任务
  • 支撑切换时机
  • 落脚点补偿
  • 接触力分配

如果浮动基状态偏了，整个人形控制链都会一起偏。

三、IMU 在人形里到底提供了什么

IMU 是人形状态估计里最核心的传感器之一。

3.1 IMU 最直接提供什么

通常包括：

  • 角速度
  • 线加速度相关量

有些系统还会结合磁力计，但在人形和室内机器人上，很多时候磁力计并不总是可靠主力。

3.2 IMU 为什么不可替代

因为它有几个关键特性：

  1. 高频
  2. 响应快
  3. 不依赖外部环境纹理
  4. 对动态姿态变化非常敏感

这对人形特别重要，因为人形最怕在：

  • 单支撑
  • 落脚
  • 扰动恢复

这些高动态阶段“感知慢半拍”。

3.3 但 IMU 单独并不够

原因也非常明确：

  • 陀螺仪会漂
  • 加速度积分会累积误差
  • 纯惯导长期位置一定会漂

所以在人形里，IMU 是状态估计主干，但不是完整答案。

四、编码器在人形状态估计里扮演什么角色

关节编码器虽然不能直接告诉你浮动基在哪里，但它依然非常关键。

4.1 编码器最擅长什么

它擅长提供：

  • 关节位置
  • 关节速度
  • 下肢构型

4.2 为什么这和浮动基状态有关

因为一旦某只脚稳定接地，人形下肢链条就能把：

  • 支撑脚
  • 膝
  • 髋
  • 骨盆

之间的几何关系连起来。

也就是说，编码器加运动学，能帮你把“接触到的脚”和“浮动基主刚体”联系起来。

4.3 编码器的问题是什么

它的问题不是没用，而是：

  • 只能告诉你关节怎么动
  • 不能单独决定身体相对世界怎么动
  • 受传动间隙和结构柔性影响

所以编码器本身不是全状态，但在接触建立后，它会变成极其关键的约束信息来源。

五、足底接触为什么会成为状态估计的核心约束

这一点是人形状态估计最值得真正吃透的地方之一。

5.1 接触不是一个简单标签

接触常常被简化理解成：

  • 0：没接地
  • 1：接地

这当然是一层信息，但在人形估计里，接触的价值远不止如此。

5.2 一旦脚稳定接地，会发生什么

如果某只脚真的稳定接地，那么这只脚相对于世界的速度在接触方向上就应当接近零。

这意味着：

它不只是“接触了”，它还给系统提供了运动学约束。

5.3 这类约束有什么用

它可以帮助你限制浮动基估计漂移。

因为如果支撑脚不动，而编码器告诉你髋、膝、踝现在是什么姿态，那么通过运动学链就能推回浮动基相对这只脚的状态。

5.4 为什么这在人形里特别关键

因为人形本来就是浮动基系统。

一旦没有外部定位，支撑脚接触约束就是最重要的“世界锚点”之一。

六、单支撑和双支撑对状态估计意味着什么

这不是只对控制有影响，对估计也同样关键。

6.1 双支撑阶段

如果两只脚都稳定接地，那么系统理论上有更多约束可用。

好处是：

  • 浮动基恢复更稳
  • 姿态和速度估计约束更强

坏处是：

  • 两只脚未必都是真正“刚性稳定接地”
  • 若实际接触质量不一致，盲目把两脚都当硬约束会引入冲突

6.2 单支撑阶段

约束数量少了，但结构更干净。

这时系统主要依赖：

  • 单脚接触约束
  • IMU 动态信息
  • 编码器运动学链

6.3 估计器最怕什么

最怕的是：

  • 你以为双支撑，其实其中一只脚已经在滑或虚接触
  • 你以为单支撑已建立，其实支撑转移还没完成

这会让估计器把错误接触关系当成硬约束，从而把浮动基状态拉偏。

所以人形估计里，“接触是否成立”绝不是细枝末节，而是核心逻辑。

七、为什么人形状态估计比四足更脆

四足也依赖 IMU 和足端接触约束，但双足通常更脆。

7.1 支撑裕度更小

双足一旦接触判断错或姿态估计偏一点，失稳代价通常更直接。

7.2 支撑切换更激烈

双足频繁经历：

  • 双支撑到单支撑
  • 单支撑到双支撑

估计器必须跟得上这种切换。

7.3 上身扰动更敏感

人形的手臂、躯干、头部动作都可能显著扰动整机状态。

这让 IMU 观测和运动学约束之间更容易出现复杂耦合。

7.4 一个现实判断

同样的估计误差，在机械臂上可能只是末端偏一点，在双足上可能直接变成“下一步没接住自己”。

八、一个典型的人形状态估计链长什么样

如果从工程实现角度看，一个比较典型的估计链通常包含这些输入：

  1. IMU
  2. 关节编码器
  3. 足底力传感器或接触状态
  4. 有时还有视觉、外部定位或运动捕捉

8.1 IMU 提供高频姿态动态信息

尤其是：

  • 角速度
  • 短时姿态变化
  • 动态响应

8.2 编码器提供关节构型

让你知道此刻下肢链条是什么姿态。

8.3 足底接触提供世界约束

把“脚”和“世界”关系钉住，帮助恢复浮动基。

8.4 视觉或外部定位提供长期漂移校正

如果系统有这些传感器，它们通常能帮助缓解长期漂移。

但在人形基础行走控制里，很多时候最关键的还是：

IMU + 编码器 + 接触约束。

九、为什么很多系统会在加速度层或误差状态层做估计

这是人形估计里比较常见的一种实现思路。

9.1 原因一：动力学和控制本身就在加速度层高频运行

所以把状态更新和约束修正在这个层面组织起来，接口更自然。

9.2 原因二：误差状态更适合处理小偏差修正

在姿态、速度、偏置这类量上，很多系统更喜欢估计“误差”而不是完全重建全量状态。

9.3 工程意义是什么

这通常能带来：

  • 数值上更稳定
  • 更适合高频递推
  • 更容易和 IMU 偏置估计结合

这里不展开某一种滤波器细节，但你需要建立一个明确认识：

人形状态估计不是简单做个平滑，而是带动态模型和接触约束的系统估计。

十、IMU 偏置为什么会成为长期隐患

人形控制里，IMU 的价值很大，但偏置问题同样不能低估。

10.1 陀螺仪偏置会带来什么

它会让姿态积分逐渐漂。

短时间看不明显，时间一长就会出现：

  • 躯干姿态慢慢歪
  • 估计角速度与真实不一致

10.2 加速度偏置会带来什么

如果拿它参与速度和位置恢复，会导致：

  • 线速度漂移
  • 浮动基位置越来越不准

10.3 为什么偏置在人形里更危险

因为人形对姿态和速度估计非常敏感。

一点点长期漂移，在单支撑、转向、扰动恢复这些动作里，可能会被显著放大。

所以成熟系统往往都会认真处理：

  • IMU 零偏标定
  • 在线偏置估计
  • 长期漂移纠正

十一、足底传感器和接触开关的价值到底差在哪

这在硬件设计里很实际。

11.1 接触开关

优点：

  • 简单
  • 成本低
  • 接触事件检测直接

缺点：

  • 信息量少
  • 只能告诉你“像是接触了”
  • 无法直接告诉你接触质量和受力分布

11.2 足底力/力矩传感器

优点：

  • 能提供法向力信息
  • 能帮助判断接触是否稳定
  • 能辅助估计压力中心

缺点：

  • 成本高
  • 标定和安装更复杂
  • 噪声和漂移处理也更难

11.3 一个务实判断

如果只是做基础原型，接触开关也能起步。

但如果目标是做更稳的人形估计与控制，足底受力信息的价值会非常大。

十二、为什么“接触成立”不等于“接触可作为约束使用”

这是一个非常容易被忽略的细节。

12.1 接触刚建立时可能并不稳定

例如：

  • 脚刚轻轻碰到地
  • 但还没有真正承担支撑
  • 或者还在滑

这时如果估计器立刻把它当成强约束，可能会把系统拉偏。

12.2 真正能作为约束的接触通常要满足什么

至少要看：

  • 法向力是否足够
  • 接触是否持续一段时间
  • 是否存在明显滑移
  • 当前支撑关系是否和控制逻辑一致

12.3 这说明估计器还要处理什么

估计器不只是融合传感器，还要做：

接触可信度判断。

这件事在人形里尤其重要。

十三、为什么很多“控制器发疯”的问题，其实是估计链断了

这个经验非常值钱。

13.1 典型现象

比如：

  • 站立还行，一抬脚就晃
  • 落脚瞬间大幅摆动
  • 手一伸出去，整机姿态突然不合理
  • 转向时总往一边偏

第一反应往往是去调：

  • QP 权重
  • 轨迹参数
  • 力矩分配

但真正的问题可能是：

  • 机身姿态估计偏了
  • 支撑脚判断错了
  • 浮动基速度估计飘了

13.2 为什么这种错觉常见

因为控制器是最后表现症状的地方。

它拿到错状态，输出当然会乱。

所以做人形调试时，必须长期保持一个习惯：

先怀疑估计链，再怀疑控制律。

这通常比反过来更高效。

十四、状态估计和全身控制是怎么闭环起来的

这条线闭环的关键就在这里。

14.1 估计器输出什么

估计器给出：

  • 浮动基姿态
  • 浮动基速度
  • 接触状态
  • 关节状态

14.2 全身控制器用这些量做什么

它用来：

  • 计算质心状态
  • 判断当前支撑结构
  • 构建接触约束
  • 组织任务目标
  • 求解接触力和控制输出

14.3 控制器反过来又影响什么

控制器输出的行为会改变：

  • 关节运动
  • 足底接触
  • 机身姿态
  • 支撑切换

于是新的状态又回到估计器里。

14.4 所以真正的闭环是什么

不是“估计器给控制器一组数字”这么简单，而是：

估计链决定控制器看到的世界，控制器又持续改变估计链要解释的身体状态。

这才是人形系统真正的估计-控制闭环。

十五、一个更接近真实系统的估计周期

15.1 读入 IMU

拿到高频角速度和加速度信息。

15.2 读入关节编码器

更新当前全身构型。

15.3 读入足底接触信息

判断：

  • 哪只脚在支撑
  • 哪只脚可能滑
  • 哪些接触约束当前可信

15.4 利用运动学链和接触约束修正浮动基状态

把脚和骨盆之间的关系重新钉回世界参考。

15.5 更新偏置和协方差或内部置信度

这一步决定估计器下一周期“更信谁”。

15.6 输出给控制器

包括：

  • 姿态
  • 速度
  • 接触状态
  • 有时还包括质心近似状态和足底受力相关量

这就是为什么估计器也必须是实时系统的一部分。

十六、从仿真到真机时，状态估计最该先查什么

16.1 IMU 安装方向和坐标系

这是最低级但最致命的问题之一。

16.2 时间戳同步

IMU、编码器、足底接触不同步，会直接导致估计错位。

16.3 接触判断阈值

阈值太松或太紧，都会导致接触状态误判。

16.4 足底滑移现象

如果脚在滑，但估计器还把它当刚性支撑，问题会很严重。

16.5 偏置漂移

长时间运行后如果姿态和速度越来越偏，首先就该怀疑偏置处理。

十七、如果自己做人形状态估计，推荐推进顺序

第一步：先把姿态估计做稳

先不要急着恢复完整世界位置。

第二步：再把接触判断做稳

确认哪只脚真的能作为可信支撑约束。

第三步：再做浮动基速度恢复

让系统先知道自己大概“怎么在动”。

第四步：再做浮动基位置恢复

这一步对长期漂移更敏感，也更依赖约束质量。

第五步：最后再和全身控制强耦合验证

比如：

  • 单支撑
  • 落脚
  • 行走中操作

这样推进会比一开始就追完整全状态更稳。

十八、核心结论

18.1 人形状态估计的核心，不是“平滑传感器”，而是恢复浮动基和接触状态

这是本质区别。

18.2 IMU 是主干，但不能单独解决全部状态问题

它必须和编码器、接触约束结合。

18.3 足底接触在人形里不是附加信息，而是世界参考的重要来源

没有它，浮动基恢复会非常脆。

18.4 接触是否成立，和接触是否能作为可信约束，是两件事

这一点要严格区分。

18.5 很多看起来像控制器问题的失稳，本质上是状态估计链先出错了

这个判断非常重要。

18.6 人形真正的闭环，是估计和控制共同定义整机行为

不是先估计完、再控制这么简单。

十九、总结

人形机器人中的状态估计，真正解决的不是“把几个传感器拼起来”，而是：

在一个浮动基、多接触、持续切换支撑关系的系统里，持续恢复一个足够真实、足够稳定、足够能用于控制的身体状态。

这也是为什么人形控制越往深处走，越会发现：

  • 平衡离不开估计
  • 步态离不开估计
  • 全身控制离不开估计
  • 接触力分配离不开估计

可以压缩成一句话：

人形机器人不是靠控制器单独站起来的，而是靠状态估计先把“自己此刻到底是什么样”这件事尽可能说对，控制器才有机会把它接住。
