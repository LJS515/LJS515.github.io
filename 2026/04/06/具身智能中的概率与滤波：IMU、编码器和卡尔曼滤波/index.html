具身智能中的概率与滤波：IMU、编码器和卡尔曼滤波

机器人一旦上真实硬件，很快就会遇到一个非常现实的问题：

机器人并不知道自己“准确地”在哪里，也不知道自己“准确地”在怎么动。

为什么？

因为机器人看到的、测到的、算出来的，几乎全部都有误差。

比如：

  • IMU 会漂
  • 编码器会抖
  • 电机有背隙
  • 相机会受光照影响
  • 深度数据会有空洞
  • 足端触地判断会有延迟

所以一旦机器人从仿真进入现实，你就必须正视一件事：

现实世界不是确定的，机器人必须学会在不确定里工作。

下面把这件事讲清楚。

一、为什么机器人一定离不开概率和滤波

概率统计看起来像基础数学课内容，但在机器人里非常贴近工程。

但事实上，在机器人里，概率论最核心的用途不是“算概率题”，而是：

表示不确定性。

这非常关键。

因为机器人系统中的很多量，你都无法直接获得“绝对真值”。

你拿到的通常只是：

  • 一个带噪声的测量值
  • 一个带误差的估计值
  • 一个会逐渐漂移的积分结果

所以机器人真正面对的问题不是：

“这个值是多少？”

而是：

“这个值大概是多少，我对它有多确定？”

这就是概率和状态估计进入机器人的入口。

这就是滤波。

三、机器人里最常见的两类误差

很多误差问题，先不要急着上公式，先分类会更清楚。

3.1 随机噪声

这类误差的特点是：

  • 看起来在抖
  • 每次不完全一样
  • 通常围绕某个值上下波动

比如：

  • 编码器量化噪声
  • IMU 瞬时抖动
  • 力传感器微小波动
  • 相机检测框的小幅晃动

3.2 系统误差

这类误差更麻烦，因为它不是纯随机，而是“带方向”的。

比如：

  • IMU 零偏
  • 相机外参没标准
  • 轮径建模不准
  • 齿轮间隙
  • 结构安装偏差

它们会让系统长期偏离真实值。

3.3 为什么系统误差更危险

因为随机噪声通常还能靠平均、滤波来减弱。

但系统误差如果不校准，会一直把系统往错误方向带。

所以工程里一个非常重要的认知是：

滤波不是万能补丁，标定和建模错误不能全靠滤波挽救。

四、IMU 到底是什么，它能告诉机器人什么

IMU 是机器人里极其常见的一类传感器。

IMU 通常包含：

  1. 加速度计
  2. 陀螺仪
  3. 有时还会有 磁力计

4.1 加速度计测什么

它测的是“比力”，工程里通常近似理解为加速度相关量。

它可以帮助你估计：

  • 倾斜方向
  • 线加速度变化
  • 动态响应

4.2 陀螺仪测什么

它测角速度。

也就是：

  • roll 方向在多快地转
  • pitch 方向在多快地转
  • yaw 方向在多快地转

4.3 IMU 为什么这么重要

因为它有几个很强的优点：

  1. 频率高
  2. 响应快
  3. 不依赖外部环境
  4. 对高速动态特别敏感

这使得 IMU 在下面这些系统里极其关键：

  • 四足机身姿态估计
  • 人形平衡控制
  • 机械臂末端振动观测
  • VIO 和 SLAM
  • 无人机姿态稳定

五、编码器到底在干什么

编码器是机器人里另一类基础传感器。

它最核心的作用是：

告诉你关节或者电机轴转了多少。

5.1 编码器擅长什么

它非常擅长测：

  • 角位置
  • 角速度
  • 累计转动量

所以在机械臂、腿式机器人、灵巧手中，编码器通常是关节状态最核心的来源。

5.2 编码器的优势

  1. 与关节直接相关
  2. 位置测量通常比较稳定
  3. 适合闭环控制

5.3 编码器的局限

它也有很多限制：

  1. 只能测“关节发生了什么”
  2. 不能直接知道机器人在世界坐标系下的位置
  3. 会受传动间隙、柔性变形影响
  4. 速度通常是差分算出来的，容易抖

所以编码器很好，但它不够完整。

因为：

  • 陀螺仪有零偏
  • 加速度计有偏置
  • 数值积分会累积误差

所以 IMU 很适合提供：

  • 高频动态信息
  • 短时间姿态变化

但不适合单独长期给出稳定绝对位置。

这就是为什么必须做融合。

七、滤波到底在解决什么问题

“滤波”这个词听起来像是在做信号处理，其实在机器人里它更像一种状态估计思想。

你可以把它理解成：

在噪声和不确定性存在的前提下，估计系统真实状态。

这个状态可能包括：

  • 位置
  • 速度
  • 姿态
  • 偏置
  • 接触状态

7.1 滤波不是单纯“让曲线变平滑”

这是个常见误解。

虽然移动平均、低通滤波确实可以让曲线变平滑，但机器人里的滤波通常不止如此。

更核心的问题是：

  • 当前状态是多少
  • 哪个传感器更可信
  • 模型预测和观测冲突时该信谁

八、先区分三种常见滤波思路

8.1 简单平滑滤波

比如：

  • 移动平均
  • 一阶低通
  • 中值滤波

这类方法简单、便宜、易实现，适合处理高频抖动。

但它们的局限也明显：

  • 没有显式系统模型
  • 延迟较大
  • 处理复杂动态能力有限

8.2 互补滤波

互补滤波特别适合 IMU 姿态估计入门。

它的核心思想是：

  • 陀螺仪短时间动态准，但会漂
  • 加速度计长期方向信息稳定，但瞬时噪声大

那就把两者结合：

  • 高频部分更多相信陀螺仪
  • 低频部分更多相信加速度计

这就是“互补”。

对很多嵌入式姿态项目来说，这是非常实用的第一步。

8.3 卡尔曼滤波类方法

这是更系统化的状态估计方法。

它把下面几件事统一起来了：

  1. 系统如何演化
  2. 传感器如何观测
  3. 过程噪声有多大
  4. 测量噪声有多大

然后根据这些信息动态更新状态估计。

九、卡尔曼滤波为什么这么经典

卡尔曼滤波之所以经典，不是因为它“高级”，而是因为它非常符合机器人的真实需求。

它特别适合解决这样的问题：

  1. 系统有动态模型
  2. 观测有噪声
  3. 希望递推估计状态

这几乎就是机器人状态估计的标准场景。

9.2 卡尔曼滤波本质上在干什么

一句话概括：

在“模型预测”和“传感器观测”之间做最优折中。

十、协方差为什么这么重要

如果只抓住卡尔曼滤波里的一个关键词，应该是：

协方差。

因为滤波器真正判断“该更相信谁”的依据，核心就是不确定性。

10.1 什么是协方差矩阵

你可以先把它理解成：

对状态误差大小和相关性的描述。

比如：

  • 某一维误差大，说明这一维不确定
  • 某两维误差相关，说明它们会一起偏

10.2 过程噪声和测量噪声

卡尔曼滤波里有两类非常重要的噪声：

  1. 过程噪声
    模型本身不完美带来的不确定性
  2. 测量噪声
    传感器观测不准带来的不确定性

工程上常写成：

  • Q：过程噪声协方差
  • R：测量噪声协方差

10.3 为什么调 Q 和 R 本质上是在调“信谁”

如果 R 很大，说明你觉得测量很不靠谱。

那滤波器就会更相信模型预测。

如果 Q 很大，说明你觉得模型也不太可靠。

那滤波器就会更依赖实时测量。

所以很多时候调滤波器，并不是在“神秘炼丹”，而是在显式表达：

你对系统模型和传感器的信任程度。

十二、IMU 和编码器融合，为什么特别常见

因为它们刚好是互补的。

12.1 IMU 的特点

  • 高频
  • 快
  • 短时动态好
  • 长期会漂

12.2 编码器的特点

  • 关节位置稳定
  • 适合闭环
  • 与本体关节直接相关
  • 不能单独提供完整全局状态

12.3 两者融合的意义

你可以粗略理解成：

  • IMU 提供“怎么在快速变化”
  • 编码器提供“关节实际上转了多少”

在四足、人形、机械臂、云台等系统里，这种融合都很常见。

比如在腿式机器人上：

  • IMU 提供机身姿态变化
  • 编码器给出关节配置
  • 再结合足端接触信息，去估计机身状态

这已经是实际机器人状态估计的核心套路之一。

十三、为什么四足和人形对状态估计尤其敏感

机械臂很多时候固定在底座上，状态估计相对简单一些。

但四足和人形不是。

它们本体会移动、会摆动、会接触切换，所以状态估计更难，也更关键。

13.1 四足机器人

四足状态估计常常要解决：

  • 机身姿态
  • 机身速度
  • 机身位置
  • 哪只脚在接触地面

这里 IMU 非常关键，但单靠 IMU 又不够。

因为脚一旦稳定接地，足端相对于地面某种程度上就变成了“约束观测”。

这时编码器和运动学也能帮助状态估计。

13.2 人形机器人

人形更复杂，因为：

  • 接触切换更频繁
  • 平衡要求更高
  • 摔倒代价更大

所以姿态估计、足底接触判断、质心估计都特别重要。

13.3 一个很现实的结论

很多腿式机器人的问题，表面上看像“控制不好”，其实往往有相当一部分根源在：

状态估计不准。

控制器再强，如果输入状态本身是歪的，也很难救。

十四、视觉和滤波的关系

很多人提到滤波，只想到 IMU 和编码器。

其实视觉系统同样大量依赖滤波与状态估计。

14.1 视觉检测会抖

比如目标检测框、关键点、位姿估计结果通常都不是稳定不动的。

如果直接把它们送给控制器，机械臂末端可能就会跟着抖。

这时通常就需要：

  • 平滑
  • 跟踪
  • 多帧融合

14.2 VIO 和 SLAM 更是滤波重灾区

视觉惯导里程计和 SLAM 里，核心问题就是：

  • 视觉有观测
  • IMU 有高频动态
  • 两者都不完美

于是系统必须持续估计：

  • 位姿
  • 速度
  • 偏置

这本质上就是更复杂的状态估计问题。

十五、扩展卡尔曼滤波为什么会出现

前面讲的经典卡尔曼滤波，最适合线性系统。

但机器人里很多系统不是线性的。

例如：

  • 旋转姿态
  • 非线性运动学
  • 三角函数关系

所以就有了扩展卡尔曼滤波（EKF）。

15.1 EKF 在做什么

它的思路大致是：

  • 真实系统是非线性的
  • 那就在当前工作点附近做线性化
  • 再按卡尔曼滤波思路更新

也就是说，它不是把非线性消灭，而是：

在局部用线性近似处理。

15.2 为什么 EKF 在机器人里这么常见

因为机器人里到处都是非线性。

例如：

  • 姿态更新
  • 关节到末端映射
  • 足端接触约束
  • 视觉投影模型

所以很多实际系统会用：

  • EKF
  • Error-State EKF
  • UKF
  • 因子图优化

但对入门来说，先把普通卡尔曼滤波直觉吃透最重要。

十六、你在工程里真正需要关注什么

这一节我尽量讲得更像做项目，而不是上理论课。

16.1 采样频率

传感器频率不同，会直接影响滤波效果。

比如：

  • IMU 200Hz
  • 编码器 1kHz
  • 相机 30Hz

不同频率数据怎么同步，本身就是工程问题。

16.2 时间戳

如果时间戳不准，再好的滤波器也会出问题。

尤其是：

  • 视觉 + IMU
  • 多关节同步
  • 网络分布式系统

16.3 坐标系统一

滤波不是脱离几何存在的。

如果 IMU 数据在一个坐标系下，编码器推的状态在另一个坐标系下，却没正确转换，融合结果一定不对。

16.4 噪声参数标定

Q 和 R 最好不是瞎猜。

至少应该通过：

  • 静态采样
  • 多次实验
  • 数据统计

来大致估计噪声水平。

16.5 不要迷信某一种滤波器

工程上最好的方法，不一定是最复杂的方法，而是：

在你的系统频率、噪声水平、计算资源、实时性约束下最合适的方法。

很多小型嵌入式系统，互补滤波已经很好用。

很多复杂机器人系统，EKF 或因子图才更合适。

关键是问题匹配。

十七、初学者最容易踩的坑

17.1 把滤波当成“万能修复器”

不是。

如果：

  • IMU 装反了
  • 编码器方向反了
  • 外参没标准
  • 时间同步乱了

那滤波器不可能神奇地把一切修好。

17.2 只追求“曲线好看”

曲线平滑不等于状态估计正确。

有时候你把低通滤得太狠，曲线是好看了，但延迟很大，控制反而更差。

17.3 不区分噪声和偏置

随机噪声和系统偏置是两种问题。

如果不区分，调滤波器会越调越乱。

17.4 一上来就用复杂滤波器

如果简单移动平均、互补滤波都没搞清楚，直接上 EKF，通常会学得很虚。

17.5 忽略调试可视化

状态估计一定要多画图：

  • 原始传感器
  • 预测值
  • 融合值
  • 误差趋势

没有可视化，调滤波器非常痛苦。

十八、你应该怎么学这部分最有效

这里给你一个很实用的学习顺序。

第一步：先观察原始数据

先不要急着滤波。

先采一段：

  • IMU 静态数据
  • 编码器静态数据
  • 机器人慢速运动数据

看它们长什么样。

第二步：做最简单的平滑

比如：

  • 均值滤波
  • 一阶低通

目的不是解决全部问题，而是建立噪声直觉。

第三步：做互补滤波

尤其适合姿态估计入门。

第四步：做一维卡尔曼滤波

先做：

  • 位置 + 速度
  • 或角度 + 角速度

这种简单系统。

第五步：再进入多传感器融合

比如：

  • IMU + 编码器
  • IMU + 视觉
  • IMU + 足端接触

这个路线会比一上来直接啃复杂状态估计论文更扎实。

十九、核心结论

这一节只保留最关键的工程判断。

19.1 机器人面对的不是“真值”，而是“带误差的观测”

这是状态估计的起点。

19.2 IMU 快，但会漂

它擅长高频动态，不擅长长期绝对定位。

19.3 编码器稳，但信息不完整

它能很好描述关节状态，但不能单独解决全部状态估计问题。

19.4 滤波的本质是融合“预测”和“观测”

不是单纯让曲线变好看。

19.5 卡尔曼滤波的核心是按不确定性加权融合

谁更可信，取决于模型和测量噪声。

19.6 真正好的状态估计，是几何、时间同步、标定、传感器和滤波一起工作的结果

不是某个公式单独拯救系统。

二十、总结

具身智能中的概率与滤波，归根到底不是“数学炫技”，而是机器人进入真实世界后的生存能力。

因为一旦离开理想仿真，机器人就必须面对：

  • 噪声
  • 漂移
  • 不确定性
  • 传感器冲突

而状态估计做的事情，就是在这些不完美里尽可能恢复一个稳定、可信、可用于控制的内部世界模型。

可以压缩成一句话：

机器人不是先“看见真实”，再去行动，而是在噪声和误差中一边估计、一边行动。
